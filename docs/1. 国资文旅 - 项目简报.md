# 国资文旅 - 项目简报

## 项目背景
国资文旅是继**国资商城**和**国资云农**后的又一个上海团队负责的大项目，一期目标是在 5 月 20 号开发出正式上线的 iOS 和 Android 版本，保证流程走通、体验完整。

## 人员分工
上海前端团队主要负责 App 端开发，云南团队负责后台开发。
上海前端团队分**架构组**和**业务组**，**架构组**负责框架优化、通用组件、App 封装 & 对接等相关工作；**业务组**负责具体业务的开发、联调和上线。
国资文旅**业务组**成员各自负责的模块如下：
* **王建阳**：玩法+收银台
* **王建辉**：注册登录+常用旅客
* **申飞**：首页
* **钱丹**：我的订单+我的点评+项目列表+项目详情
* **岳乃杉**：我的主页+个人资料
* **姚恒**：营位预订+房车预订
* **李冬梅**：房车列表&详情+修改密码+专题H5页面
* **刘欢**：相册+精品酒店

具体任务安排请查看 Jira，每个人负责的业务模块多多少少都会和其它人的模块有所关联，如果开发中遇到业务关联的问题，记得找到对应的人进行沟通。

## 开发须知
这里主要着重强调一下开发流程，避免大家走穹路浪费时间。

### 开发准备
1. 配置开发环境，node 统一使用稳定版本 6.9/6.10，建设安装 nvm 用来管理 node 版本。
2. 安装必要工具
	* Git 工具：**Tower**(Mac)/**TortoiseGit**(Windows)
	* API 调试：**Paw**(Mac)/**Postman**(Windows)
	* 代码对比：**Kaleidoscope**(Mac)/**Beyond Compare**(Windows)
	* 上面的工具如果有更好的选择，欢迎推荐～
3. 开发之前一定认真阅读 3 个文件：效果图、原型图 & 交互图。
	* 页面样式以**效果图**为准。
	* 页面交互参照**交互图**。
	* 边界情况 & 特殊说明都在**原型图**里有标注。

### 开发中
1. 请参照 B2B App 项目，国资文旅会在其框架基础上搭建，很多页面在 B2B 中已经做过，新入职的同学可以多参考，提高开发效率。
2. 根据后端定好的 API 来组织路由和页面。
3. GitLab 操作规范
	1. 每个人都要先拉一个分支出来，在上面开发自己负责的页面。
	2. 分支开发完后要先发 **Merge Request**，代码审核通过后才能并到 dev。
	3. pull 之前要保证本地的新代码已经 push 或 stash，push 之前记得写 comments。

### 开发完成
1. 到 Jira 上拖任务。
2. 找到对应的后端进行联调。
3. 跟进测试，及时修复相应的 bug。
